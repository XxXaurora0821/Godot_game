# 时间回廊（暂定名）

一个 2D 像素风平台跳跃游戏（Godot），核心玩法是“操控时间”来解谜与闯关。

## 当前目标（Demo）

做出一个可完整游玩的 demo 关卡，验证这三件事：

- 时间能力是否真的好玩
- 关卡是否能围绕时间机制产生解谜深度
- 故事反转是否能让玩家记住这个项目

## 游戏类型

- 2D Platformer
- Pixel Art
- Engine: Godot

## 核心玩法（时间操控）

玩家可对“自己”和“场景对象”施加时间状态：

- 加速：平台移动更快、机关提前触发、敌人进入高频动作
- 减速：危险窗口变宽、跳跃时机更容易掌控
- 正向流动：恢复正常时间推进
- 反向流动：部分对象回到之前状态（桥恢复、门重关、敌人回位）
- 静止：短时间冻结对象（子弹、陷阱、巡逻敌人）

## 表现重点（非常关键）

时间能力必须“看起来就不一样”：

- 画面：色调变化、拖影、扭曲、扫描线/钟表UI提示
- 动效：不同时间状态对应独立粒子和屏幕反馈
- 音频：变速、倒放、低通处理，让玩家闭眼也能分辨状态
- 交互反馈：对象身上显示可被时间影响的标记（比如发光边框）

## Demo 关卡结构（建议）

- A段：教学区（逐个教会时间能力）
- B段：组合挑战（两种以上能力叠加）
- C段：终点拿到关键道具
- Return段：原路返回，但机关逻辑变化，玩家要“反向理解”同一地图

## 背景故事（Demo版）

- 主角是一个小孩，被一位老头委托去送某件东西
- 老头教会主角控制时间的能力
- 主角一路闯关，在关卡尽头拿到关键物品
- 主角被要求原路返回，证明自己真正掌握了时间能力
- 若主角继续向前，他最终会变成那个老头
- 真相：老头其实就是未来的主角自己（时间闭环）

## 美术方向（待定）

- 关键词：
- 角色：小孩（轻快） vs 老头（沉稳、神秘）
- 场景：同一区域在不同时间状态下有明显视觉差异


## Brainstorm 池（持续补充）

- 时间能力是否有资源限制（能量条/冷却）？
- 反向流动是“全局”还是“对象级”？
- 玩家能否看到“过去残影”来辅助解谜？
- 是否加入“时间悖论惩罚”（滥用能力会导致世界异常）？
- Return段是否让同一机关出现完全不同解法？
